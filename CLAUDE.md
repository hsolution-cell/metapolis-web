# 커밋 규칙

- 커밋 메시지는 **한국어**로 작성한다.
- 제목은 한 줄 요약(명사형 종결: "~ 추가", "~ 변경", "~ 수정"). 필요하면 본문에 변경 이유·내용을 짧게 적는다.
- **작업 단위별로 바로 커밋**한다 — 기능/수정 하나가 끝나면 몰아두지 말고 그때그때 커밋.
- 커밋 전에 `npm run build`(또는 최소 `npx tsc --noEmit`) 통과를 확인한다.
- `git push`는 사용자가 요청할 때만 한다.
- `.env.local` 등 비밀값·키는 절대 커밋하지 않는다.
- SQL 마이그레이션 파일(`supabase/*.sql`)은 해당 기능 커밋에 함께 포함한다.
